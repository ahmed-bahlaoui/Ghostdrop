# GhostDrop — AI Agent Context

## Project Overview

GhostDrop is an anonymous, temporary, encrypted file-sharing platform inspired by Wormhole and SendAnywhere.

Users:

* upload files
* receive a human-friendly transfer code
* retrieve files on another device using the code

The system is designed around:

* temporary storage
* E2E encryption
* ephemeral transfers
* streaming uploads/downloads
* object storage architecture

→ Fastify API
→ PostgreSQL (metadata)
→ Redis (ephemeral state / rate limiting / realtime)
→ MinIO (binary object storage)

# Database

Current main table:

* transfers

Fields include:

* id
* code
* object_key
* original_filename
* mime_type
* size_bytes
* download_count
* max_downloads
* expires_at
* created_at

Migration system implemented manually using SQL migration files and migrate.ts.

Looking into switching to an ORM like drizzle or Prisma

---

# Important Architectural Decisions

* Files are **NOT** stored in PostgreSQL.
* PostgreSQL stores **only metadata**.
* MinIO stores encrypted binary objects.
* Object keys are generated by backend.
* Transfer codes must be **human-friendly** and easy to type manually.
* Transfers are ephemeral and expire **automatically** redis ttl.

# Coding Conventions

* Use TypeScript everywhere.
* Keep services separated by responsibility.
* Prefer clear architecture over premature abstractions.
* Prefer streaming over buffering for file handling.
* Avoid large monolithic files.
* Keep infrastructure reusable.
* Adopt a modular approach
